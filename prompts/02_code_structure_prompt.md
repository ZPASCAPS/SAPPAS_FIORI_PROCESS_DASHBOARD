# 02. Code Structure Prompt

이 프로젝트는 SAP Fiori / SAPUI5 기반 대시보드 프로젝트이다.

코드는 기능별로 명확하게 분리해서 작성한다.

Cursor를 사용하지 않는 사람도 프로젝트 구조를 쉽게 이해하고 유지보수할 수 있도록 파일명, 폴더명, 주석을 명확하게 작성한다.

## 코드 작성 기본 원칙

- main 파일은 최대한 간소화한다.
- 하나의 파일에 모든 기능을 몰아넣지 않는다.
- 기능별로 view, controller, model, service, util, css 파일을 분리한다.
- 불필요한 파일을 생성하지 않는다.
- 사용하지 않는 샘플 코드, 테스트 파일, 의미 없는 컴포넌트는 만들지 않는다.
- 기존 파일 구조를 먼저 파악한 뒤 필요한 파일만 수정하거나 생성한다.
- 파일 이름만 봐도 어떤 기능인지 알 수 있게 작성한다.

## 주석 작성 원칙

각 파일의 맨 위에는 반드시 해당 파일의 역할을 주석으로 작성한다.

주석 설명을 위한 별도 파일은 만들지 않는다.

모든 설명은 해당 코드 파일 안에 작성한다.

## JavaScript 파일 상단 주석 예시

```js
/**
 * MainDashboard.controller.js
 *
 * 역할:
 * - 메인 대시보드 화면의 전체 이벤트를 제어한다.
 * - 왼쪽 모듈 탭, 중앙 프로세스 흐름, 오른쪽 챗봇 패널을 연결한다.
 *
 * 주요 기능:
 * - 초기 데이터 로딩
 * - 모듈 탭 클릭 이벤트 처리
 * - 프로세스 단계 선택 이벤트 처리
 * - 선택된 기능에 따른 상세 정보 표시
 */
