# 05. Cursor Workflow Prompt

이 프로젝트를 수정할 때는 아래 방식으로 작업한다.

## 작업 방식

1. 먼저 현재 프로젝트 폴더 구조를 분석한다.
2. 기존 파일이 어떤 역할을 하는지 파악한다.
3. 새 파일 생성이 필요한지 먼저 판단한다.
4. 불필요한 파일은 만들지 않는다.
5. 기존 파일 수정으로 해결 가능한 경우 기존 파일을 우선 수정한다.
6. 기능별 분리가 필요한 경우에만 새 파일을 생성한다.
7. 어떤 파일을 수정할지 먼저 설명한 뒤 코드를 작성한다.

## 코드 작성 방식

- 코드에는 반드시 기능 설명 주석을 작성한다.
- 각 파일 상단에는 파일 역할을 설명하는 주석을 작성한다.
- 중요한 함수에는 함수 목적과 동작 방식을 주석으로 작성한다.
- 주석만 따로 모아둔 파일은 만들지 않는다.
- main 파일은 간단하게 유지한다.
- main 파일은 전체 연결 역할만 담당하게 한다.
- 세부 기능은 별도 view, controller, model, service, util 파일로 분리한다.

## 응답 방식

코드를 작성하기 전 아래 순서로 설명한다.

1. 수정할 파일 목록
2. 각 파일의 역할
3. 생성할 파일이 필요한 이유
4. 삭제하거나 만들지 않아도 되는 파일
5. 최종 코드

## 금지 사항

- 불필요한 샘플 파일 생성 금지
- 의미 없는 테스트 파일 생성 금지
- 사용하지 않는 컴포넌트 생성 금지
- 하나의 controller에 모든 로직 몰아넣기 금지
- 실제 SAP 접속 정보, 비밀번호, API Key 하드코딩 금지
- 더미데이터와 실제 SAP OData 호출 로직을 섞어서 작성 금지

## Cursor에게 요청할 때 기본 문장

아래 문장을 기준으로 작업한다.

현재 프로젝트 구조를 먼저 분석한 뒤, SAP Fiori 통합 프로세스 대시보드에 필요한 파일만 수정하거나 생성해줘.

각 파일 상단에는 해당 파일의 기능을 주석으로 작성하고, 중요한 코드 로직에도 설명 주석을 넣어줘.

main 파일은 최대한 간소화하고, 기능별로 view, controller, model, service, util 파일을 분리해서 Cursor를 사용하지 않는 사람도 유지보수하기 쉽게 만들어줘.

불필요한 파일은 생성하지 말고, 생성이 필요한 파일은 왜 필요한지 먼저 설명해줘.

메인 대시보드는 왼쪽 모듈별 탭, 중앙 SAP 프로세스 순서도, 오른쪽 챗봇 패널 구조로 만들어줘.
